## 🚀 System Features
This system includes all mandatory features required for the final system check:
* **Basic CRUD Functionality:** Full Create, Read, Update, and Delete for student records.
* **Search:** Filter students by name, email, or course via a search form.
* **Pagination:** Limits display to 5 students per page for better readability.
* **Bootstrap UI:** Styled with Bootstrap 5 for a professional look.
* **Soft Deletes:** Records are hidden rather than permanently removed from the database.
* **REST API:** A JSON endpoint to fetch student data programmatically.

## 🛠️ Mandatory Git Workflow
As per lab requirements, development followed a strict collaborative workflow:
1. **Feature Branching:** Each member worked on a dedicated branch.
2. **Pull Requests:** Features were reviewed and merged to `main` by the Team Leader.
3. **Clean History:** Maintaining a clear commit history per member.
